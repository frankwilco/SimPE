The Pie Menu Editor Plugin (TTAB)

 

The TTAB is where you decide which options are available to a Sim to interact with the object, or for a player to set options on that object via the selected Sim.  The actual text that will appear on the menu is held in the Pie Menu Strings (TTAs) file with the matching instance number.   See the String Editor help file for details of how to edit the TTAs.

 

Headers

These values refer to the TTAB file as a whole

Filename Any descriptive title you like
Format Not sure.  Probably best not to edit this.
Commit File Updates the package file in memory to include edits to current resource.  No change made to file on disk.  If you don't commit, you can abandon edits to current resource by simply opening another resource.

.

Buttons

Add Appends a line which is an exact copy of the selected line, including all settings and BHAV calls
Delete Deletes selected line and renumbers later lines to close the gap
Append Pops up a list of available TTABs and appends all lines from the chosen TTAB to the current TTAB.   A dialog is offered with the highest Pie String ID (TTAs line number) that is referenced in current TTAB, which you can accept or edit.   Accepting is recommended unless you know you have as yet unused pie strings with a higher number that you intend to use.  During the appending, the appended lines have their Pie String ID incremented so that they reference lines in the Pie Menu String file with a value that is increased by that number.   This allows you to append the TTAs that match the appended TTABs and still have the TTAB entries referencing the appropriate text.

.

Settings tab

This is where you define what options are available, and to which Sims.

The following table details some of the commonly used fields.  In many cases there is a helpful dropdown list so you can pick from valid values:

Pie String ID This should match the number of a string in the TTAs file that has an Instance number matching the current TTAB file
Action BHAV Contains the Instance number of the BHAV you want to run when a Sim or player picks this interaction
Guardian BHAV The instance number of a BHAV that should run to test if the interaction should be available to a particular Sim.  This is an optional field, you do not need a test if the interaction is to be available to all Sims.
Attenuation Code Attenuation reduces the attractiveness of an interaction over a distance.  The higher the attenuation the closer the Sim will have to be before he feels like using the object.
Attenuation Value This value in this field is taken into account when you have selected "Custom" for the Attenuation code
Autonomy This is a threshold.  In practice, when you have Free Will turned on in the game, Sims can autonomously choose any interaction with an Autonomy value of 0x0032 or lower.  We usually put 0x0064 in this field if we don't want Sims to be able to ever choose the interaction for themselves, and then they can only do it if a player tells them to.  But it is possible a cheat may be introduced in the future which will allow Sims to reach this autonomy level, so it may need reviewing.
Join Index The number of the pie menu line that will run if another Sim tries to join the Sim running current interaction.  "Joinable" needs to be ticked in Flags for this value to be used.
The bits from UI Display Type onwards No idea!

Visitor Tick if visiting Sims can use it.

Immediately The action will be executed without the currently selected Sim having to stop his current interaction.  Obviously this can only be used for menu items that do not require the Sim to do anything (for example a light coming on, or a door being unlocked)
Consecutive The Sim can choose to do this several times in a row.

Motives tab

* NB TTAB format 0x54 was new with Pets and the pets' motives panel is different to the human one.   We do not yet know how to label the various data.

The values entered here decide how attractive a Sim will find the option - how likely he is to use it under various circumstances.  For example, if you are creating a new chair, and you give the "Sit" option a Bladder minimum value of 100 (0x0064), the Sim will think sitting on that chair will relieve his full bladder even better than most toilets.   He'll keep sitting on it over and over again until he finally wets himself

The columns are grouped by the type of Sim they apply to.  If you tick "Show all" you would have the chance to enter different values for every age group.  In practice this is rarely done, and usually you would set them for Adults, then use "Set all like this" to copy the whole columns to the other ages.   Then usually you just make sure the ages (and species) you don't want using the option are not ticked on the Settings tab.

 

  • Minimum is the least level of that is advertised to all Sims for this motive.

  • Delta is the amount extra that would be advertised to Sims with a particular attribute, on a sliding scale according to how much of that attribute they have.

  • Type is the Sim’s attribute that matters when deciding whether to advertise the extra.
